🤗 smolagents CHEAT SHEET

Hugging Face's minimal library for agents that think in code.

Core idea: the LLM writes Python snippets to call your tools instead of emitting JSON tool calls. ~1,000 lines of core code, model-agnostic.

📋 Source article ↗ Docs ↗ Repo ↗

👋 Hello, agent

from smolagents import CodeAgent, InferenceClientModel
model = InferenceClientModel()          # default HF Inference model
agent = CodeAgent(tools=[], model=model)
print(agent.run("Sum the numbers from 1 to 10"))

🧩 Agent types

Two flavors — pick by how the agent should act:

  • CodeAgent — writes Python that composes tool calls. Best for reasoning & composition.
  • ToolCallingAgent — emits structured (JSON) tool calls. Best for reliable atomic calls.
from smolagents import ToolCallingAgent, WebSearchTool
agent = ToolCallingAgent(tools=[WebSearchTool()], model=model)

🛠️ Common constructor args

agent = CodeAgent(
    tools=[...],              # list of Tool INSTANCES (not classes)
    model=model,
    add_base_tools=True,      # include the default toolbox
    max_steps=10,             # cap reasoning steps
    verbosity_level=1,        # 0 silent, 1 steps, 2 detailed
    planning_interval=3,      # re-plan every N steps (None = off)
    stream_outputs=True,
    additional_authorized_imports=["numpy", "pandas"],  # CodeAgent only
    executor_type="local",    # local|e2b|docker|blaxel|modal|wasm
    step_callbacks=[my_callback],
    name="...", description="...",  # only if used as a sub-agent
)

▶️ Running

agent.run("task")                                # final answer
agent.run("task", reset=False)                   # keep memory (follow-up)
agent.run("task", additional_args={"k": value})  # inject objects/state
agent.run("Describe this", images=[pil_image])   # vision input
for chunk in agent.run("task", stream=True): ... # stream steps

🧰 Tools

Default toolbox ([toolkit] extra)

WebSearchTool · DuckDuckGoSearchTool · GoogleSearchTool · VisitWebpageTool · PythonInterpreterTool · FinalAnswerTool · UserInputTool · SpeechToTextTool

Add them all with add_base_tools=True.

Custom tool — @tool decorator (quick)

Needs type hints + a docstring with an Args: section — these become the schema the LLM sees.

from smolagents import tool
@tool
def get_product_price(product_name: str) -> float:
    """Gets the price of a product.
    Args:
        product_name: The name of the product to look up.
    """
    return PRICES[product_name]
agent = CodeAgent(tools=[get_product_price], model=model)

🔗 Tool collections & MCP

# From a Hub collection (tools from all its Spaces)
from smolagents import ToolCollection, CodeAgent
coll = ToolCollection.from_hub("huggingface-tools/...",
                               trust_remote_code=True)
agent = CodeAgent(tools=[*coll.tools], add_base_tools=True, model=model)

# From an MCP server (stdio)
from mcp import StdioServerParameters
params = StdioServerParameters(command="uvx", args=["mcp-server-fetch"])
with ToolCollection.from_mcp(params, trust_remote_code=True) as tc:
    agent = CodeAgent(tools=[*tc.tools], model=model)
    agent.run("Fetch the homepage of huggingface.co")

# From an MCP server (HTTP)
with ToolCollection.from_mcp(
    {"url": "http://127.0.0.1:8000/mcp",
     "transport": "streamable-http"},
    trust_remote_code=True,
) as tc:
    agent = ToolCallingAgent(tools=[*tc.tools], model=model)

🤖 Multi-agent orchestration

Give a sub-agent a name + description, pass it via managed_agents. The manager calls it like any tool.

from smolagents import (CodeAgent, ToolCallingAgent,
                        WebSearchTool, VisitWebpageTool)

web_agent = ToolCallingAgent(
    tools=[WebSearchTool(), VisitWebpageTool()],
    model=model,
    name="web_search_agent",
    description="Runs web searches. Give it your query as an arg.",
)

manager = CodeAgent(
    tools=[], model=model,
    managed_agents=[web_agent],
    additional_authorized_imports=["time", "numpy", "pandas"],
)
manager.run("Who is the CEO of Hugging Face, and their birth year?")

🛡️ Secure code execution

CodeAgent runs locally by default (restricted interpreter: your tools + a safe builtins/imports list). For untrusted workloads, sandbox it:

agent = CodeAgent(..., executor_type="e2b")     # needs E2B_API_KEY
agent = CodeAgent(..., executor_type="docker")
agent = CodeAgent(..., executor_type="blaxel")  # BL_API_KEY + BL_WORKSPACE

⚠️ The LLM can generate arbitrary code. Never authorize unsafe imports on untrusted input; prefer a sandbox in production.

🔍 Inspect, debug, replay

agent.memory.steps   # full record: task, thoughts, calls, timings, tokens
agent.replay()       # pretty re-print of the whole run
agent.logs           # raw logs
Step callback (runs after each step)
from smolagents import ActionStep, CodeAgent
def log_step(step: ActionStep, agent: CodeAgent) -> None:
    print(f"Done step {step.step_number}")
    if step.step_number > 5:
        step.observations_images = None   # drop images, save tokens
agent = CodeAgent(..., step_callbacks=[log_step])
Drive the loop manually
from smolagents import TaskStep, ActionStep
agent.memory.steps.append(TaskStep(task="...", task_images=[]))
final = None; n = 1
while final is None and n <= 10:
    s = ActionStep(step_number=n, observations_images=[])
    final = agent.step(s)
    agent.memory.steps.append(s); n += 1

⚡ Quick decisions

  • CodeAgent vs ToolCallingAgent → composition/reasoning vs reliable atomic calls.
  • Which model class → HF Inference: InferenceClientModel · OpenAI/Anthropic/etc.: LiteLLMModel · any OpenAI-compatible URL: OpenAIServerModel · local: TransformersModel.
  • Custom tool → simple fn: @tool · stateful/complex: subclass Tool.
  • Production → low max_steps, sandbox (e2b/docker), never trust raw imports, keep a human in the loop.
