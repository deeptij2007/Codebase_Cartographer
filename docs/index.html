🗺️ Codebase Cartographer CHEAT SHEET

An agentic onboarding tool built on smolagents. Point it at any unfamiliar git repo and it writes an onboarding map — architecture, dependency diagram, load-bearing modules, a git-history risk map, and a "read these first" path.

Python 3.10+ smolagents multi-agent AST + git mining GitHub ↗

⚡ Quickstart

# 1 — install
pip install -r requirements.txt

# 2 — configure a backend
cp env.example .env
set -a; source .env; set +a

# 3 — run (from the dir that CONTAINS
#     the Codebase_Cartographer/ folder)
python -m Codebase_Cartographer.main /path/to/repo

Output lands in a folder named after the analyzed repo: repo/ONBOARDING.md + a Mermaid diagram.

🤖 Agent hierarchy

cartographer (manager · CodeAgent) tools: read_file imports: json, textwrap max_steps: 12 │ ├── structure_mapper (CodeAgent) max_steps: 14 imports: ast, collections, json, re, math, itertools, networkx, pandas │ walks the tree · parses ASTs · builds the import graph · emits the Mermaid diagram │ └── git_historian (ToolCallingAgent) max_steps: 10 mines git history for churn hotspots & bus-factor / knowledge-concentration risk

The manager delegates to both specialists (wired via managed_agents=[...]), verifies a few files itself, then synthesizes one Markdown report.

🧰 Guarded tools

  • Structure (mapper)
  • list_files · read_file · file_stats
  • detect_entrypoints · build_import_graph
  • export_dependency_diagram
  • History (historian)
  • run_git · git_churn_hotspots · git_authorship

No raw os/subprocess: every read & git call goes through these, staying inside the repo and capping output size.

📄 The report it produces

  • What this system does — plain-English summary
  • Architecture at a glance — layers & data flow
  • Dependency diagram — embedded Mermaid
  • Core modules — the load-bearing files
  • Risk & fragility map — churn + bus-factor
  • Start here — ordered ~6-file reading path

🛡️ Safety notes

  • CodeAgent executes model-written code with a local executor by default. For untrusted repos, sandbox it — add executor_type="docker" (or "e2b") in agents.py.
  • The import graph & entry-point detection are heuristics (Python-only graph) — great for orientation, not a substitute for reading the code.
  • Tool output is capped (default 6000 chars) so a huge file can't flood the context window.
